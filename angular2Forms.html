Forms? Easy peacy

Just make an input tag, then a submit button

and we are done... what can possibly go wrong?

Some things that need to be considered...

  • Input is dangerous, it modifies data
  • It can be entered in funny ways
  • Expectations and Error Messages need to be clear
  • Parts of the form can have complex dependencies
  • As forms can be heavily UI reliant, they are hard to test

Controls and Control Groups

A control represents a single input field. It encapsulates the fields value and state

Control States

NgControl automatically watches for changes. Controls have values and a validation state which is determined by an optional validation function

StateClass (if True)Class (if false)
Has been visitedng-touchedng-untouched
Value changedng-dirtyng-pristine
Value validng-validng-invalid

Of course you must add the css needed to reflect the state
